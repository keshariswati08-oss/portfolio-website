Generative AI
GenAI for Business
GenAI handles unstructured content, generates outputs, and reasons through ambiguity. For BA/PM: AI requirements need model accuracy thresholds, hallucination controls, fallback processes, and explainability requirements — especially critical in FCA and RBI-regulated environments. Standard BRD templates miss all of this.
Applied: Built 4 production AI agents on Anthropic API with defined system prompts, JSON output schemas, and governance documentation.

Multiple AI models orchestrated into autonomous workflows — each agent with a defined role, passing structured outputs to the next. Architecturally identical to the 5-bot ARTP chain I designed as BA, but with AI reasoning at each step. New BA questions: handoff conditions, confidence thresholds, chain failure escalation, human override triggers.
Applied: Designed and shipped 4-agent AI toolkit with inter-agent handoffs, structured outputs, and Notion API integration.
Prompt Engineering
Requirements for AI Models
Prompt engineering is BA requirements applied to AI — precision of instruction determines quality of output. In practice: defining the AI's role, output schema, edge case handling, constraint logic, and tone. I write production system prompts the same way I write BRDs — with acceptance criteria, fallback conditions, and version control.
Applied: 4 production system prompts with JSON schemas, edge case handling, and versioned in Git alongside product decision docs.
